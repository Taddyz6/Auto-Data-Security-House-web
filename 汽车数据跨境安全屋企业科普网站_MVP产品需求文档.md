# 汽车数据跨境安全屋企业科普网站
## MVP 产品需求文档（可直接交付 TREA 开发）

**文档版本：** V1.0  
**产品阶段：** 最小可行产品（MVP）  
**面向对象：** 产品、设计、前端、后端、测试及 TREA 开发智能体  
**建议技术路线：** Next.js / React + TypeScript + Tailwind CSS + SQLite 或 Supabase  
**首版目标：** 用一个可访问、可交互、可留资的网站，让汽车企业在 5 分钟内理解“数据跨境安全屋是什么、如何工作、是否适合自身业务”，并完成需求自测与咨询预约。

# 1. 项目概述

## 1.1 项目背景

国内不同地区、自由贸易试验区和行业的数据跨境制度存在差异，境外不同国家和地区也分别适用不同的数据保护、网络安全、汽车网络安全和行业监管规则。

整车企业、零部件企业、动力电池企业普遍存在跨国研发、海外路试、车联网运营、售后诊断、供应链协同、电池碳足迹等数据跨境需求，但企业通常无法直观理解以下问题：

1. 数据跨境安全屋与普通云、VPN、跨境专线、数据出境安全评估之间有什么区别；
2. 企业数据进入安全屋后，内部经过哪些识别、审核、传输、监测和存证步骤；
3. 哪些数据可以直接传输，哪些需要审批、脱敏、备案或禁止出境；
4. 国内规则与境外规则如何在同一套流程中进行匹配；
5. 安全屋是否适合企业当前的数据类型、业务目的地、数据规模和合规现状。

本网站不直接承担真实数据跨境传输，而是作为汽车数据跨境安全屋的企业科普、能力展示、需求诊断和咨询获客入口。

## 1.2 产品定位

面向汽车产业的数据跨境安全屋科普与企业需求预诊断平台。

网站需要完成四件事：

- **讲清楚：** 用通俗语言解释数据跨境安全屋；
- **展示清楚：** 用交互流程展示数据进入安全屋后的完整处理逻辑；
- **判断清楚：** 通过轻量问卷帮助企业初步判断适配场景和关注事项；
- **承接需求：** 收集企业咨询、演示和参观预约信息。

## 1.3 MVP 核心目标

首版不追求完整法规数据库、真实 AI 合规判断或真实跨境传输，而应优先验证：

1. 企业是否能够看懂安全屋；
2. 企业最关注哪些跨境场景；
3. 自测工具是否能够形成有效咨询线索；
4. 网站是否能够支撑对外演示和线下宣讲；
5. 哪些页面和内容值得进入下一阶段深度建设。

## 1.4 非目标

MVP 阶段明确不建设：

- 真实企业数据上传与解析；
- 真实 AI 数据分类分级模型；
- 真实法规自动判定引擎；
- 真实跨境传输链路；
- 真实区块链上链；
- 企业账户体系；
- 在线支付；
- 复杂工单系统；
- 多租户企业后台；
- 政府监管驾驶舱。

以上能力可在后续版本中建设。首版只做前台展示、模拟交互、信息采集和后台内容维护。

# 2. 用户与使用场景

## 2.1 目标用户

### 用户 A：整车企业合规或数据安全负责人

关心国内外法规、重要数据识别、个人信息出境、审计材料和处罚风险。

### 用户 B：研发、IT 或车联网负责人

关心系统如何接入、支持哪些文件和接口、传输效率、审批方式、网络架构和安全措施。

### 用户 C：零部件或电池企业管理人员

缺乏专职合规团队，希望快速判断当前跨境业务是否需要专业服务。

### 用户 D：政府、园区、协会及合作机构

需要了解项目能力、试点价值、应用场景和示范效果。

## 2.2 核心用户路径

### 路径一：首次了解

首页进入 → 阅读安全屋定义 → 查看与普通云/专线区别 → 浏览四大能力 → 查看完整流程 → 提交咨询。

### 路径二：带着业务问题进入

首页选择业务场景 → 查看场景数据类型和管控流程 → 进入企业自测 → 获得初步建议 → 留下联系方式。

### 路径三：线下宣讲辅助

工作人员打开网站 → 播放安全屋流程演示 → 切换不同数据场景 → 展示安全基础设施 → 展示案例与资质。

# 4. 页面需求

## 4.1 首页

### 4.1.1 页面目标

用户进入后 30 秒内理解：

- 安全屋解决什么问题；
- 安全屋与普通传输方式的区别；
- 安全屋包含哪些核心能力；
- 下一步可以查看流程或进行自测。

### 4.1.2 首屏内容

主标题：

> 汽车数据跨境安全屋  
> 让汽车数据跨境流动可控、合规、可监测、可追溯

副标题：

> 面向跨国研发、海外路试、车联网运营、电池碳数据和全球供应链场景，提供数据识别、合规预审、安全传输、风险监测和存证审计的一体化能力展示。

主按钮：

- 查看完整流程
- 企业需求自测

首屏右侧建议为动态流程示意图，不使用复杂视频。可采用自动循环动画：

企业数据 → AI 识别 → 合规预审 → 审批/放行 → 安全传输 → 实时监测 → 存证报告

### 4.1.3 什么是安全屋

使用一句话定义：

> 汽车数据跨境安全屋，是部署在企业数据与境外接收方之间的受控处理与传输环境。数据在出境前经过识别、分类、规则校验、审批和安全处理，传输过程中持续监测，传输结果形成完整记录和审计证据。

### 4.1.4 四类方案对比

以对比卡片或表格展示：

| 对比项 | 普通云盘 | VPN/跨境专线 | 数据出境咨询 | 数据跨境安全屋 |
|---|---|---|---|---|
| 文件传输 | 支持 | 支持 | 不支持 | 支持 |
| 数据识别 | 通常不支持 | 不支持 | 人工分析 | 规则与智能识别 |
| 出境前审核 | 不支持 | 不支持 | 阶段性人工审核 | 流程化预审 |
| 风险监测 | 有限 | 主要监测网络 | 不持续 | 全链路监测 |
| 审批留痕 | 有限 | 有限 | 文档记录 | 全流程留痕 |
| 审计存证 | 通常不支持 | 通常不支持 | 输出咨询材料 | 自动形成证据链 |
| 适用场景 | 普通协作 | 已明确合规的数据传输 | 前期合规评估 | 持续、高频、复杂跨境业务 |

页面中必须说明：本对比为通俗化说明，实际能力以具体项目方案和部署范围为准。

### 4.1.5 四大核心能力

四张可点击卡片：

1. **智能合规预审**  
   识别数据类型、敏感字段和业务场景，匹配适用规则，输出初步风险和处置建议。

2. **安全可控传输**  
   通过审批、权限、加密、身份验证和专属链路控制数据流转。

3. **全链路合规检测**  
   对文件、接口、日志和跨境流量进行持续监测，识别异常传输和违规风险。

4. **可信存证与审计**  
   留存主体、时间、数据类型、审批、传输和处置记录，支持审计报告和证据核验。

### 4.1.6 典型场景入口

五个场景卡片：

- 跨国联合研发
- 海外路试与测试
- 海外车联网运营
- 动力电池碳足迹
- 全球供应链协同

点击进入相应场景详情页或场景弹窗。

### 4.1.7 首页底部转化区

文案：

> 不确定企业当前的数据能否跨境？  
> 用 3 分钟完成初步需求自测。

按钮：

- 开始自测
- 预约专家沟通

## 4.2 认识安全屋页面

### 4.2.1 页面结构

1. 为什么需要安全屋
2. 安全屋解决的核心问题
3. 安全屋内部由什么组成
4. 安全屋不是什么
5. 常见问题

### 4.2.2 为什么需要安全屋

使用三类问题说明：

- 规则多：境内、境外、行业和地区要求同时存在；
- 数据杂：图纸、代码、日志、地图、个人信息、碳数据格式不同；
- 链路散：专线、云接口、邮件、网盘、车端上行等路径难以统一管理。

### 4.2.3 五层能力示意

展示五个组成部分：

1. 安全基础设施；
2. AI 合规引擎；
3. 跨境传输系统；
4. 合规检测系统；
5. 存证审计系统。

每个部分点击后显示 80—150 字解释。

### 4.2.4 安全基础设施示意

简化展示以下组件：

- 防火墙；
- 隔离网闸；
- SD-WAN 合规链路；
- 流量探针；
- 云桌面；
- 堡垒机；
- 日志审计；
- 综合监测平台。

交互方式：鼠标悬停显示用途，移动端点击显示。

### 4.2.5 安全屋不是什么

明确避免误解：

- 不是只提供一条网络专线；
- 不是只做一次性数据出境咨询；
- 不是把数据全部上传到公共云；
- 不是自动保证所有数据都可以出境；
- 不是以技术系统替代企业法务和合规责任。

## 4.3 全流程演示页面

### 4.3.1 页面目标

用模拟数据让企业直观看到一份数据从提交到完成跨境的全过程。

### 4.3.2 流程步骤

完整流程固定为：

1. 企业发起跨境任务；
2. 填写业务场景和目的地；
3. 数据接入；
4. AI 数据类型与敏感字段识别；
5. 国内外规则匹配；
6. 合规预审；
7. 人工复核或审批；
8. 脱敏、裁剪或拦截；
9. 加密传输；
10. 流量与行为监测；
11. 境外接收方身份验证；
12. 完整性校验；
13. 日志存证；
14. 输出中英文审计记录。

### 4.3.3 交互方式

采用横向步骤条或纵向时间线。

每个步骤需包含：

- 步骤名称；
- 通俗说明；
- 主要输入；
- 系统动作；
- 可能输出；
- 风险点。

### 4.3.4 模拟演示

提供四个预设样例，用户选择后流程内容动态变化：

#### 样例 A：整车研发图纸

模拟结果：

- 初步识别：研发设计数据；
- 风险提示：可能包含商业秘密或受限技术信息；
- 建议动作：确认数据范围、审批权限、接收方和用途；必要时进行字段裁剪或文件分级；
- 展示状态：进入人工复核。

#### 样例 B：车辆轨迹数据

模拟结果：

- 初步识别：位置轨迹和车辆运行数据；
- 风险提示：可能涉及个人信息、敏感个人信息或重要数据；
- 建议动作：确认采集范围、数据规模、是否脱敏、是否具有合法处理依据；
- 展示状态：高风险，暂停并复核。

#### 样例 C：电池碳足迹数据

模拟结果：

- 初步识别：电池供应链和碳排放数据；
- 风险提示：注意供应商商业信息、数据来源和境外使用范围；
- 建议动作：核验字段最小化、接收方权限和合同约束；
- 展示状态：满足条件后可进入审批。

#### 样例 D：售后故障日志

模拟结果：

- 初步识别：车辆诊断和售后服务数据；
- 风险提示：日志中可能包含 VIN、账号、位置、设备标识；
- 建议动作：移除非必要标识，按项目和时效授权；
- 展示状态：脱敏后复核。

### 4.3.5 重要提示

所有模拟结果页面必须显示：

> 本演示仅用于说明安全屋工作流程，不构成正式法律意见或数据出境合规结论。实际判断需结合数据内容、规模、主体、目的地、接收方、处理目的和最新适用规则进行专业评估。

## 4.5 企业需求自测

## 4.5.1 产品原则

自测不是法律判断工具，而是需求分流和咨询线索收集工具。

不得输出“合规”或“不合规”的确定性结论，只输出：

- 初步适配程度；
- 重点关注事项；
- 推荐下一步；
- 推荐咨询服务。

## 4.5.2 表单步骤

采用多步骤表单，共 6 步。

### 第一步：企业基本信息

字段：

- 企业名称：选填；
- 企业类型：必填；
  - 整车企业
  - 零部件企业
  - 动力电池企业
  - 科技/平台企业
  - 检测认证机构
  - 其他
- 联系人姓名：结果页留资时必填；
- 手机或邮箱：结果页留资时至少填写一个。

### 第二步：业务场景

多选：

- 跨国研发；
- 海外路试；
- 海外车联网运营；
- 售后诊断；
- 供应链协同；
- 电池碳足迹/电池护照；
- 数据分析或模型训练；
- 其他。

### 第三步：跨境方向

字段：

- 数据方向：
  - 中国境内传向境外；
  - 境外传向中国；
  - 双向传输；
- 目的地：
  - 欧盟；
  - 英国；
  - 美国/加拿大；
  - 日本/韩国；
  - 东南亚；
  - 中东；

- 是否涉及多个国家：是/否。

### 第四步：数据类型

多选：

- 研发图纸和技术文件；
- 软件代码和算法；
- 高精地图或地理数据；
- 视频、图像、点云；
- VIN 和车辆标识；
- 位置轨迹；
- 车主或驾驶员信息；

### 第五步：现状与规模

字段：

- 传输频率：
  - 偶发；
  - 每月；
  - 每周；
  - 每日；
  - 实时；
- 单次数据量：
  - 小于 100MB；
  - 100MB—10GB；
  - 10GB—1TB；
  - 大于 1TB；
  - 不确定；
- 当前方式：
  - 邮件；
  - 公共网盘；
  - 企业网盘；
  - VPN；
  - 跨境专线；
  - 云接口/API；
  - 车联网平台；
  - 尚未开始；
- 是否已有审批流程：是/否/不清楚；
- 是否已有数据分类分级：是/否/不清楚；
- 是否保留完整传输日志：是/否/不清楚。

### 第六步：主要困难

多选：

- 国内外规则难以同时满足；
- 人工审核慢；
- 数据量大、格式复杂；
- 缺少安全传输通道；

- 审计和举证困难；
- 缺乏专业人员；
- 需要支持临时项目；
- 其他。

## 4.5.3 评分逻辑

MVP 使用前端规则计算，不调用 AI。

定义三个维度：

### A. 业务复杂度

- 单一场景：1 分；
- 两个场景：2 分；
- 三个及以上场景：3 分；
- 多国家或双向传输：额外 2 分；
- 每日或实时传输：额外 2 分。

### B. 数据敏感关注度

以下每选一项加 2 分：

- 高精地图或地理数据；
- 位置轨迹；
- 车主或驾驶员信息；
- 生物识别；
- 视频、图像、点云；
- 软件代码和算法。

以下每选一项加 1 分：

- VIN；
- 车辆日志；
- 研发图纸；
- 电池与碳数据；
- 生产和供应链数据。

### C. 治理成熟度缺口

以下每项为“否”或“不清楚”加 1 分：

- 分类分级；
- 审批流程；
- 完整日志。

当前方式包含公共网盘或邮件，加 2 分。

## 4.5.4 结果分类

总分 0—5：

> **基础咨询型需求**  
> 当前场景相对集中，建议优先完成数据清单梳理、主体与目的地确认，并建立基础审批和日志留存机制。

总分 6—10：

> **流程建设型需求**  
> 企业已存在较明确的跨境业务，建议开展场景化数据盘点、规则比对、分类分级、审批流程和安全传输方案设计。

总分 11 分及以上：

> **安全屋重点适配型需求**  
> 企业涉及多场景、多类型或高频数据跨境，建议进一步评估智能识别、合规预审、专属传输、持续监测和审计存证的一体化安全屋方案。

## 4.5.5 结果页内容

结果页包括：

1. 自测结果类型；
2. 企业选择的业务摘要；
3. 3—5 条重点关注事项；
4. 推荐能力模块；
5. 推荐下一步；
6. 留资表单；
7. “下载自测简报”按钮。

MVP 下载简报可使用浏览器打印为 PDF，或由前端生成 HTML 打印页面，不需要服务端 PDF 引擎。

## 4.5.6 留资字段

必填：

- 联系人；
- 手机或邮箱至少一个；
- 同意隐私声明。

选填：

- 企业名称；
- 职务；
- 咨询说明；
- 期望沟通方式；
- 是否希望预约演示；
- 是否希望实地参观。

提交成功后显示：

> 信息已提交。工作人员将根据您填写的业务场景进行后续联系。

# 16. 最终产品判断标准

首版网站上线后，一名不了解安全屋的汽车企业用户应当能够：

1. 在 30 秒内说出安全屋的基本作用；
2. 在 3 分钟内理解数据进入安全屋后的主要流程；
3. 找到与自身业务最接近的场景；
4. 在 5 分钟内完成自测；
5. 获得一份非法律结论性质的初步建议；
6. 提交咨询、演示或参观需求。

满足以上六点，即认为 MVP 达到可用标准。
